# 02 – Documentação

## Problema no código ruim
- Não explica o que calcula
- Não indica unidade, taxa ou fórmula

## Por que o código bom é melhor
- Documenta o propósito, a fórmula e os parâmetros
- Facilita uso correto e manutenção

## O que foi aplicado
- JSDoc (descrição, parâmetros e retorno)
- Valores padrão e exemplo de uso
