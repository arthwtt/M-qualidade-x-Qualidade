# 03 – Padronização

## Problema no código ruim
- Mistura de estilos (nomes, espaços, casing)
- Uso de `var` sem necessidade
- Comparação frouxa `==`

## Por que o código bom é melhor
- Mantém consistência e reduz bugs
- Código fica previsível e fácil de revisar

## O que foi aplicado
- `const`/`let` no lugar de `var`
- Funções e nomes em padrão consistente
- `===` para comparação estrita
