# 01 – Legibilidade

## Problema no código ruim
- Nomes genéricos (f, a, b, c, x)
- Campos confusos (p, s)
- Falta de intenção clara

## Por que o código bom é melhor
- Nomes significativos tornam a regra de negócio óbvia
- Condições separadas deixam a leitura mais simples

## O que foi aplicado
- Renomeação de variáveis e funções
- Estruturação das condições
- Comentário curto explicando propósito
