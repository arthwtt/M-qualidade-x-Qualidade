# 08 – Eficiência

## Problema no código ruim
- Complexidade O(n²) por causa de loops aninhados
- Fica lento com listas grandes

## Por que o código bom é melhor
- Complexidade O(n) usando `Set`
- Melhor performance e mais simples

## O que foi aplicado
- Estrutura de dados adequada (`Set`)
- Loop único com early return
